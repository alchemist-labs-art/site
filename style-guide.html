Grid Aesthetic — Visual Language

background grid (subtle)
flask cells (bright, active)
structure
The entire viewport is covered by a uniform grid of squares.
Cell pitch: 14px (12px square + 2px gap). The gap is the page background showing through.
Grid aligns to viewport origin (0,0) — all elements snap to the grid.

background layer
Base cells render at #161616 (dark mode) — barely visible against #0f0f0f background.
~4% of cells are randomly selected as "alive" and pulse slowly between brightness 0–0.15.
Alive cells occasionally relocate: fade out → pick new position → fade in.
Animation speed: 0.003–0.01 interpolation per frame. Tick probability: 0.2%.
Effect: a quiet, breathing digital texture across the full viewport.

flask layer
The flask icon is rasterized into the grid — cells inside the SVG path light up.
Brightness range: 0.4–1.0 (much brighter than background).
Each cell has independent flicker: random target brightness, random interpolation speed.
Flip probability: 0.5–1.5% per frame. Speed: 0.015–0.05 per frame.
Effect: the flask appears to shimmer or bubble, like a living data visualization.

color mapping
All cells use a single greyscale formula: rgb(v, v, v) where
v = 22 + brightness × (224 − 22)
Brightness 0 → #161616 (dark cell), Brightness 1 → #e0e0e0 (full bright).
No color is ever introduced. The entire animation is monochrome.

state persistence
Animation state (cell positions + brightness) is saved to sessionStorage on page hide.
On revisit within the same session, the grid resumes from its last state — no jarring reset.

Usage Guidelines

Do

  • Use lowercase for all text
  • Maintain monospace typeface throughout
  • Create hierarchy with size and opacity
  • Keep borders at 1px, sharp corners
  • Use greyscale only — no color
  • Let the grid animation provide visual interest
  • Use transparent backgrounds for interactive elements

Don't

  • Introduce color or gradients
  • Use rounded corners or border-radius
  • Mix typefaces or use sans-serif
  • Use uppercase or title case for headings
  • Add drop shadows or blur effects
  • Use filled/solid buttons as default state
  • Break the monochrome grid aesthetic

Pitch Deck & Presentations

Monospace works for product UI but reads as raw/unfinished at projection scale.
For decks, investor materials, and conference slides, pair Inter (body/headlines) with Courier New (wordmark, code snippets, product names only).

Headlines & Body

Inter
Weights: 400 (body), 500 (emphasis), 600 (headlines), 700 (stats/metrics)
Free via Google Fonts. Variable weight supported.
Geometric, neutral, high x-height — reads well on screens and projectors.

Brand Signature

Courier New
Use only for: the "alkera ai" wordmark, product names ("manifold"),
code snippets, and technical callouts.
Preserves brand DNA without making slides feel like a terminal.

slide rules
Background: solid #0f0f0f (dark) or #f5f5f5 (light). No gradients.
The grid texture may be used as a subtle background on title/divider slides only — never behind body text.
One idea per slide. Generous whitespace. Left-aligned, never centered body text.
Stats: Inter 700 at large scale, greyscale only. No pie charts — use bar/line if needed.
Flask icon: top-left corner of every slide at 24px, acts as a logo mark.
Aspect ratio: 16:9. Margins: 80px minimum on all sides.

color in decks
The greyscale-only rule still applies. If you must highlight something, use --text (white/black) against --text-dim (grey).
For charts/diagrams, you may use one accent: #7a7a7a as a mid-grey differentiator — never introduce hue.

what not to do
No all-monospace slides. No terminal aesthetic for investor content.
No stock photos. No clip art. Diagrams should use the grid-cell style or simple line art.
No slide transitions or animations beyond a simple fade.

Formal Documents & Print

Contracts, white papers, proposals, and letterhead follow the same Inter + Courier New pairing.
Print defaults to light mode. Documents should feel clean and authoritative, not techy.

document typography
Title: Inter 600, 18–24pt. Body: Inter 400, 10–11pt, line-height 1.6.
Captions/footnotes: Inter 400, 8–9pt, color --text-dim.
Code/product names: Courier New inline, same size as surrounding body text.
Page headers: flask icon (16px) + "alkera ai" wordmark in Courier New, separated by a 1px rule.

layout
Paper size: A4 or US Letter. Margins: 1 inch / 25mm all sides.
Single column for letters/proposals. Two column acceptable for white papers.
Page numbers: bottom-right, Inter 400, 8pt.
No headers on the first page.

print color
Always print in light mode: #1a1a1a text on white paper.
The grid texture should not appear in print documents — it's a screen-only element.
Flask icon prints as a stroke outline in black.
