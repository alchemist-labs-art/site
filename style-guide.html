Grid Aesthetic — Visual Language

background grid (subtle)
flask cells (bright, active)
structure
The entire viewport is covered by a uniform grid of squares.
Cell pitch: 14px (12px square + 2px gap). The gap is the page background showing through.
Grid aligns to viewport origin (0,0) — all elements snap to the grid.

background layer
Base cells render at #161616 (dark mode) — barely visible against #0f0f0f background.
~4% of cells are randomly selected as "alive" and pulse slowly between brightness 0–0.15.
Alive cells occasionally relocate: fade out → pick new position → fade in.
Animation speed: 0.003–0.01 interpolation per frame. Tick probability: 0.2%.
Effect: a quiet, breathing digital texture across the full viewport.

flask layer
The flask icon is rasterized into the grid — cells inside the SVG path light up.
Brightness range: 0.4–1.0 (much brighter than background).
Each cell has independent flicker: random target brightness, random interpolation speed.
Flip probability: 0.5–1.5% per frame. Speed: 0.015–0.05 per frame.
Effect: the flask appears to shimmer or bubble, like a living data visualization.

color mapping
All cells use a single greyscale formula: rgb(v, v, v) where
v = 22 + brightness × (224 − 22)
Brightness 0 → #161616 (dark cell), Brightness 1 → #e0e0e0 (full bright).
No color is ever introduced. The entire animation is monochrome.

state persistence
Animation state (cell positions + brightness) is saved to sessionStorage on page hide.
On revisit within the same session, the grid resumes from its last state — no jarring reset.

Usage Guidelines

Do

  • Use lowercase for all text
  • Maintain monospace typeface throughout
  • Create hierarchy with size and opacity
  • Keep borders at 1px, sharp corners
  • Use greyscale only — no color
  • Let the grid animation provide visual interest
  • Use transparent backgrounds for interactive elements

Don't

  • Introduce color or gradients
  • Use rounded corners or border-radius
  • Mix typefaces or use sans-serif
  • Use uppercase or title case for headings
  • Add drop shadows or blur effects
  • Use filled/solid buttons as default state
  • Break the monochrome grid aesthetic
